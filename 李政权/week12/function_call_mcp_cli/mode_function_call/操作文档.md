# Function Call 天气助手 · 操作文档

本文说明方式一（Function Call）的命令行与 Web 前端用法，重点覆盖：**多人同时使用**、**短期记忆**、**页面会话生命周期**。

---

## 1. 前置条件

### 1.1 安装依赖

在项目根目录 `function_call_mcp_cli` 下执行：

```bash
pip install -r requirements.txt
```

Web 前端额外依赖：`fastapi`、`uvicorn`（已写入 `requirements.txt`）。

### 1.2 配置 API Key

必须设置有效的 DeepSeek Key，否则聊天会返回 401 / 503。

**PowerShell（当前窗口临时生效）：**
```powershell
$env:DEEPSEEK_API_KEY = "sk-你的密钥"
```

**永久生效（需新开终端）：**
```powershell
setx DEEPSEEK_API_KEY "sk-你的密钥"
```

申请地址：https://platform.deepseek.com/

验证是否已设置（不打印密钥内容）：
```powershell
if ($env:DEEPSEEK_API_KEY) { "OK" } else { "未设置" }
```

## 2. Web 前端操作（推荐）

每个浏览器标签页 = 一个独立会话 = 一份独立短期记忆。关闭页面后会话与记忆一并销毁。

### 2.1 启动服务

```bash
cd function_call_mcp_cli
python -m mode_function_call.web_app --host 127.0.0.1 --port 8765
```

看到类似输出即成功：
```
[web] LLM ready: deepseek-chat
[web] http://127.0.0.1:8765/
```

浏览器打开：**http://127.0.0.1:8765/**

可选参数：
| 参数 | 默认 | 说明 |
|------|------|------|
| `--host` | `127.0.0.1` | 监听地址；局域网访问可改 `0.0.0.0` |
| `--port` | `8765` | 端口 |

### 2.2 单用户对话

1. 打开页面后，右上角会出现「会话 访客-xxxx」与「记忆 0/6」
2. 在底部输入框提问，例如：`宁德今天天气怎么样？`
3. 等待回答；若调用了工具，气泡下方会显示 `get_weather(...)` 与耗时
4. 继续追问以验证短期记忆，例如：`那北京呢？`、`刚才哪个更冷？`

记忆窗口默认保留最近 **6 轮**（用户问 + 助手答算一轮）。超出后最早的轮次会被丢弃。

### 2.3 多用户模式（多开页面）

1. 再开一个浏览器窗口或标签页，访问同一地址
2. 每个页面会自动创建新会话（会话 ID 不同）
3. 右上角「在线 N」表示当前服务端存活的会话数
4. 在页面 A 问「上海天气」，在页面 B 问「那广州呢？」——B **不会**引用 A 的记忆（会话隔离）

适合课堂演示：两位同学各开一页，同时提问，互不干扰。

### 2.4 关闭会话与销毁记忆

| 操作 | 效果 |
|------|------|
| 关闭标签页 / 关闭窗口 | 浏览器 `sendBeacon` 通知服务端销毁该会话及短期记忆 |
| 刷新页面 | 先销毁旧会话，再创建新会话（记忆不会带到刷新后） |
| 异常断网 / 强杀进程 | 心跳超时约 **45 秒** 后，服务端自动清理僵死会话 |

服务端日志示例：
```
[session+] a1b2c3d4e5f6 (访客-a1b2) | active=1
[session-] a1b2c3d4e5f6 (访客-a1b2) | active=0
[session~] timeout ...          # 心跳超时清理
```

### 2.5 页面状态说明

| 角标 | 含义 |
|------|------|
| 会话 访客-xxxx | 当前页面对应的会话（悬停可看完整 session_id） |
| 记忆 x/6 | 本会话已记忆轮数 / 上限 |
| 在线 N | 服务端当前活跃会话总数 |
| 会话已失效 | 会话已被销毁，需刷新页面重建 |

### 2.6 API 一览（可选）

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/session` | 创建会话 |
| `GET` | `/api/session/{id}` | 查询会话与历史 |
| `DELETE` | `/api/session/{id}` | 销毁会话 |
| `POST` | `/api/session/{id}/close` | 供 `sendBeacon` 关闭 |
| `POST` | `/api/session/{id}/heartbeat` | 心跳续期 |
| `GET` | `/api/sessions` | 列出所有活跃会话 |
| `POST` | `/api/chat` | 提问（body: `session_id`, `question`） |

示例：
```bash
# 创建会话
curl -X POST http://127.0.0.1:8765/api/session -H "Content-Type: application/json" -d "{}"

# 提问（替换 SESSION_ID）
curl -X POST http://127.0.0.1:8765/api/chat -H "Content-Type: application/json" -d "{\"session_id\":\"SESSION_ID\",\"question\":\"杭州天气如何？\"}"
```

## 3. 命令行操作

核心脚本：`mode_function_call/run_function_call.py`  
共用逻辑：`mode_function_call/agent_core.py`

### 3.1 交互模式（默认）

```bash
python mode_function_call/run_function_call.py
```

| 输入 | 作用 |
|------|------|
| 直接提问 | 用当前用户提问 |
| `@alice 北京天气？` 或 `alice: 北京天气？` | 切换到指定用户并提问 |
| `/user bob` | 仅切换当前用户 |
| `/who` | 查看当前用户与记忆摘要 |
| `/users` | 列出已知用户 |
| `/memory` | 打印当前用户短期记忆 |
| `/clear` | 清空当前用户记忆 |
| `/clearall` | 清空全部用户记忆 |
| `/q` | 退出 |

### 3.2 常用参数

```bash
# 单次提问
python mode_function_call/run_function_call.py -q "宁德今天天气怎么样？"

# 指定用户 ID（配合 -q）
python mode_function_call/run_function_call.py -u alice -q "上海天气？"

# 单用户多轮演示（含指代追问，验证记忆）
python mode_function_call/run_function_call.py --demo

# 多人线程并发循环演示
python mode_function_call/run_function_call.py --multi --rounds 2

# 调整短期记忆轮数（默认 6）
python mode_function_call/run_function_call.py --memory-turns 8

# 供 compare.py 解析
python mode_function_call/run_function_call.py -q "..." --json --quiet
```

### 3.3 命令行多人与记忆要点

- 同一进程内，按 **用户名** 隔离记忆（与 Web 按 **session_id** 隔离类似）
- `--multi` 用线程池同时跑 alice / bob / carol，观察交错输出与互不干扰的记忆
- CLI 退出进程后，记忆随之消失（进程内短期记忆，不落盘）

## 4. 推荐演示流程

### 场景 A：Web 短期记忆

1. 启动 Web，打开一个页面  
2. 问：`宁德今天天气怎么样？`  
3. 再问：`那北京呢？`  
4. 观察第二问是否正确理解「那…呢」并查询北京  

### 场景 B：Web 多会话隔离

1. 打开页面 A、页面 B  
2. A 问：`上海和广州哪个更热？`  
3. B 问：`刚才说的城市呢？`  
4. 预期：B **无法**引用 A 的上下文（应提示缺少上文或重新询问）  

### 场景 C：关闭即销毁

1. 在某页连续聊几轮，确认「记忆 x/6」增加  
2. 关闭该页  
3. 在服务端终端应看到 `[session-]`  
4. 再开新页，「记忆」从 `0/6` 重新开始

## 6. 常见问题

**Q1：页面提示 `LLM 未配置` 或接口 503**  
未读到 `DEEPSEEK_API_KEY`。确认环境变量后**重启** `web_app`。

**Q2：`Authentication Fails ... api key is invalid`（401）**  
Key 无效或过期。到 DeepSeek 控制台换新 Key，用 `setx` 写入后新开终端再启动。

**Q3：关闭页面后「在线」数量没立刻减少**  
`sendBeacon` 一般立即生效；若浏览器限制，最多等待约 45 秒心跳超时清理。

**Q4：刷新后之前的对话没了**  
符合设计：刷新 = 关旧会话 + 开新会话，短期记忆不落盘、不跨页恢复。

**Q5：端口 8765 被占用**  
换端口：`python -m mode_function_call.web_app --port 8766`

**Q6：命令行 `from mode_function_call.agent_core import ...` 失败**  
请在项目根目录执行，保证根目录在 `PYTHONPATH` / 当前工作目录下。

**Q7：天气结果城市不对**  
后端已对地名歧义做了处理；可换写法如「宁德市」「北京市」再试。
